Node.js Introduction

Node.js is a Java Script runtime environment. This allows JavaScript to run standalone applications without the help of a browser. This allows you to use Java Script on servers as a server side scripting language. This is a very common use for the Node.js environment. There are many pro's cons to Node.js relative to PHP. One pro is that Node.js allows both server side and client side applications to be written in Java Script. This is useful for teams as it enables them to communicate in the same scripting language and even share some libraries. The usability is the main advantage as Node.js is slower than PHP in most cases.

Node.js Projects

Project #1 - Creating a Server and Hello World!

First thing is first, in order to run Node.js application you need to download it. Download it here.

Now we will create a Java Script file to run on Node.js. All node applications are written in files with the js extension. Create a new document entitled helloWorld.js. Then type the following...


var http = require('http');

This is the first line, and all it does is import a http module. Many useful functions exist within this module. Next, under the http module expression write this...


http.createServer(function (request, response){});

Here we are calling a function within the http module known as 'createServer'. This function takes a function as a parameter. The function passed to the createServer is an anonymous function and takes two parameters 'request' and 'response'. These parameters are objects whose values are filled in by the HTTP server. More on that later. Our function is not entirely written yet, this is just a step by step walkthrough. The code is not near complete. Next, between the curly braces of our anonymous function add the following...


response.writeHead(200, {'Content-Type': 'text/plain'});
response.end('Hello World\n');

The first line calls another function which is associated with the response object. This function, 'writeHead' takes two arguments. The first argument is the status code, a status code of 200 signals everything is alright. The second argument is an object which helps establish the response headers. In this case we are setting our content type to text / html.

The second line deals with a function of the response object known as end. End has three parametrs; data, encoding and callback. In this case, we are just using a string which will more or less simulate another function known as write, This will simply add the text to our page as the default encoding is UTF-8 and our callback is and does not need to be specified. Then the message will be complete and the code will end. Every response must call end.

Next, we are going to apply the listen function of the createServer object. After the closing parentheses of the http.createServer function add this...


.listen(8080);

This creates a listener on port 8080. Whenever this port is accessed our code will run.

Then, finally add a console.log statement like this at the very end.


console.log('Server started');

This will let you know the server is running after you start the application.

Now, your file in it's entirety should look like this.


var http = require('http');

http.createServer(function(request, response) {
 response.writeHead(200, {'Content-Type' : 'text/plain'});
 response.end('Hello World\n');
}).listen(8080);

console.log('Server started');

Now you can run your application by typing node helloWorld.js in the terminal. Now you know the basics, start experimenting!

Node.js References

Modules and their Content

http module

Methods

 http.createServer()

  This creates a new object of the http.Server class.

 http.request()

  This method makes a HTTP request to a server. It creates an objets of the http.ClientRequest class.

 http.get()

  This method is just like http.request() only it is more specific as it always ueses the HTTP method of GET and automatically calls req.end().

Classes

 http.Agent

  This class helps handle HTTP networking. It queues requests and manages sockets.

 http.ClientRequest

  An object of this class is created when either http.request() or http.get() is called. It represents an in-progress request whose header has already been queued.

 http.Server

  Objects of this class are commonly created when creating a server using http.createServer(). Once an object of this class is created you can access the methods of this class, such as listen().

 http.ServerResponse

  This class establishes the properties and methods of a servers response such as the end() and writeHead() methods. It forms the response objects behavior.

 http.IncomingMessage

  This class handles the behavior and properties of objects that interact with request objects from servers and response events from http.ClientRequest objects. It is also allows access to useful properties of reponses such as the status code, headers and URL.
