CSS Introduction

CSS stands for cascading style sheets. You use css files to alter the appearance of HTML files. you can change the size, position, colors and fonts of your web page with the help of CSS. CSS files take the form of selector{property:value;}. A selector can be a tag such as p, an id (prefaced with #) such as #id or a class (prefaced with .) such as .class. After the selector there is a set of curly braces. Within the curly braces you put a property, then a semi-colon and then the properties value. Each property statement should end in a semi-colon, selectors can have many property statments. See Project #1 below to get started!

CSS Projects

Project #1 - CSS Selectors

In this project we will work with three types of selectors, a tag name , an id and a class. We will be working with the following HTML document...

<!DOCTYPE html>
<html lang="en">
 <head>
  <title>
   Hello World!
  </title>
  <link rel="stylesheet" type="text/css" href="style.css">
 </head>
 <body>
  <h1 id="colorMeRed">
   Hello World!
  </h1>
  <p>
   My font is changed!
  </p>
  <p class="underlineMe">
   I'm underlined!
  </p>
  <p class="underlineMe">
   So am I! But the first paragraph is not!
  </p>
 </body>
</html>

First off, notice the link tag in the head of the HTML document. This is how you link your CSS file to your HTML file. The value of the href property should be replaced by the path to your desired CSS file.

Now we will work with the tag selector. We will target the paragraph tag like so...

p{
 font-family:arial, verdana;
}

This will affect all paragraph tags, their fonts will be changed to arial, unless that font is unavailabe for some reason. Then it will be changed to verdana as a backup. The font-family property can have many fonts seperated by commas. The fonts listed first are prioritized and will be used if possible.

Next we will work with an id selector, id selectors always begin with a '#'. The id must match the id given to an elements id attribute. For instance in the HTML document above the h1 tag has an id attribute with the value 'colorMeRed'. We would change properties of this element like so...

*note: an id can only be used for one element, overlapping id's are not good. You can give every element an id if you want, but their values should be different. If you want to affect more than one element use a class instead of an id.

#colorMeRed{
 color:red;
}

This will change the color the words "Hello World!" to red. You can also refer to a color in other ways, for instance the hex code for red is #ff0000. More on that later, right now we'll just focus on selectors.

Now we will talk about class selectors...

The last two paragraph tags both share the class 'underlineMe', we can change both of them with the same selector, like this...

.underlineMe{
 text-decoration:underline;
}

This will underline the last two paragraphs. Note how the first paragraph is not underlined as it has no class attribute. Classes and id's override tag selectors as well. And id's override classes. The more specific selector will always take precedence.
